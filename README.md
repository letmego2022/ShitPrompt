## 项目简介

`ShitPrompt` 主要折腾本地小模型得prompt：

- 当任务是**结构化判断**时，Prompt 写成论文风就一定更好吗？
- “输出理由”会提升准确率，还是只是让 AI 一边干活一边写检讨？

本仓库当前记录的是第一个实验：手机号是否属于真实有价值个人数据的过滤任务（AKA：号码到底是不是“真人料”）。

---

## 核心结论（先看这个）

- 在**负向样本**（一眼日志味、噪声味）里：
  - **不要理由**（精简 Prompt）更省 token、速度更快。
- 在**正向样本**（确实有人物信息）里：
  - **要理由**（复杂 Prompt）反而更稳，耗时也不一定更高。

👉 简单说：**该解释的时候解释，该闭嘴的时候闭嘴。**

## 实验一：要不要让模型给理由

### 对比 Prompt

```python
# ==========================================
# Prompt A（复杂版：要求输出 reason）
# ==========================================

PROMPT_A = """
任务：判断文本中的【目标手机号】是否属于真实有价值的个人数据。

判断规则：

一、必须删除（keep=false）
1. 假号码：手机号与字母连在一起（如 a1881234b）。
2. 误报：内容是程序日志、代码、错误栈、URL、系统输出。
3. 周围没有任何人物信息，仅有手机号或随机数字。

二、有价值数据（keep=true）
只要手机号附近出现以下任意信息，即视为真实个人数据：

1. 姓名（如：张三、李四）
2. 身份证号
3. 地址
4. 学校 / 班级 / 公司 / 单位
5. 年龄 / 年级
6. 家庭关系（父亲、母亲等）
7. 密码 / 账号
8. 其他可以构成人物身份的信息

示例：
手机号 + 姓名 → true
手机号 + 姓名 + 班级 → true
手机号 + 学校 → true

三、重点说明
【手机号 + 姓名】本身就已经是有价值的个人信息，必须保留。

输出要求：
只输出 JSON：

{
  "keep": true/false,
  "reason": "简要说明判断原因"
}
"""

# ==========================================
# Prompt B（精简版：不要求 reason）
# ==========================================

PROMPT_B = """
你是一个数据过滤器。

任务：
判断文本中的【目标手机号】是否属于真实个人数据。

判断规则：

keep=false：
1. 手机号与字母连在一起（如 a1881234b）
2. 内容是代码、日志、URL、系统输出
3. 只有手机号，没有任何人物信息

keep=true：
手机号附近出现以下任意信息：
- 姓名
- 身份证号
- 地址
- 学校 / 班级 / 公司 / 单位
- 年龄 / 年级
- 家庭关系
- 密码 / 账号

规则：
手机号 + 姓名 = keep=true

输出：
只返回 JSON：
{"keep": true}
或
{"keep": false}
"""
```

### 负向数据测试

**测试数据（示例）**

```python
def fake_data():
    phone = 15883380267
    return phone, "4776310;DDC15883380267C2C23021F2040B5106;..."
```

**结果**：不要理由效果更好（省 token、跑得快，主打一个不内耗）

```text
====== 对比结果 ======

Prompt A
tokens: 24269
time: 82.26s
gpu: 41°C -> 75°C

Prompt B
tokens: 19462
time: 73.59s
gpu: 57°C -> 77°C
```
