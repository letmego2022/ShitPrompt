## 项目简介

`ShitPrompt` 主要探索一个很实用的问题：

- 当任务是**结构化判断**时，Prompt 写得复杂是否一定更好？
- “输出理由”会提升准确率，还是只会增加开销？

本仓库当前记录的是第一个实验：手机号是否属于真实有价值个人数据的过滤任务。

---

## 核心结论（先看这个）

- 在**负向样本**（更像日志/噪声）里：
  - **不要理由**（精简 Prompt）更省 token、速度更快。
- 在**正向样本**（确实有人物信息）里：
  - **要理由**（复杂 Prompt）反而更稳，耗时也不一定更高。

👉 简单说：**难样本要解释，噪声样本要克制。**

## 实验一：要不要让模型给理由

### 对比 Prompt

```python
# ==========================================
# Prompt A（复杂版：要求输出 reason）
# ==========================================

PROMPT_A = """
任务：判断文本中的【目标手机号】是否属于真实有价值的个人数据。

判断规则：

一、必须删除（keep=false）
1. 假号码：手机号与字母连在一起（如 a1881234b）。
2. 误报：内容是程序日志、代码、错误栈、URL、系统输出。
3. 周围没有任何人物信息，仅有手机号或随机数字。

二、有价值数据（keep=true）
只要手机号附近出现以下任意信息，即视为真实个人数据：

1. 姓名（如：张三、李四）
2. 身份证号
3. 地址
4. 学校 / 班级 / 公司 / 单位
5. 年龄 / 年级
6. 家庭关系（父亲、母亲等）
7. 密码 / 账号
8. 其他可以构成人物身份的信息

示例：
手机号 + 姓名 → true
手机号 + 姓名 + 班级 → true
手机号 + 学校 → true

三、重点说明
【手机号 + 姓名】本身就已经是有价值的个人信息，必须保留。

输出要求：
只输出 JSON：

{
  "keep": true/false,
  "reason": "简要说明判断原因"
}
"""

# ==========================================
# Prompt B（精简版：不要求 reason）
# ==========================================

PROMPT_B = """
你是一个数据过滤器。

任务：
判断文本中的【目标手机号】是否属于真实个人数据。

判断规则：

keep=false：
1. 手机号与字母连在一起（如 a1881234b）
2. 内容是代码、日志、URL、系统输出
3. 只有手机号，没有任何人物信息

keep=true：
手机号附近出现以下任意信息：
- 姓名
- 身份证号
- 地址
- 学校 / 班级 / 公司 / 单位
- 年龄 / 年级
- 家庭关系
- 密码 / 账号

规则：
手机号 + 姓名 = keep=true

输出：
只返回 JSON：
{"keep": true}
或
{"keep": false}
"""
```

### 负向数据测试

**测试数据（示例）**

```python

    phone = 15883380267
```

## 结论解读

这组实验不是“复杂 Prompt 永远更好”或“短 Prompt 永远更好”，而是：

- **任务分布决定策略**。
- 当样本偏噪声时，`reason` 往往增加思考开销，却不带来同等收益。
- 当样本语义更完整时，`reason` 能帮助模型进行更稳定的判定。

原话总结：

> 干不成的事情，要理由真的费脑子 😅

## 可复用建议

如果你也在做分类/过滤任务，可以按下面流程快速试：

1. 先做 A/B：`有理由` vs `无理由`。
2. 分开看正负样本，不要只看总体平均。
3. 同时记录：准确率、tokens、耗时、资源占用（如 GPU 温度/显存）。
4. 最终按业务目标选策略：
   - 要省成本：优先精简 Prompt；
   - 要可解释/稳健：保留简短 `reason` 字段。
