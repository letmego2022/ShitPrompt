# ShitPrompt
ShitPrompt（屎屁）：一个关于小模型 Prompt 使用方式的有趣探索

## 测试一： “干就完了”，是否给让模型给出理由
对比提示词
```
# ==========================================
# Prompt A（复杂版）
# ==========================================

PROMPT_A = """
任务：判断文本中的【目标手机号】是否属于真实有价值的个人数据。

判断规则：

一、必须删除（keep=false）
1. 假号码：手机号与字母连在一起（如 a1881234b）。
2. 误报：内容是程序日志、代码、错误栈、URL、系统输出。
3. 周围没有任何人物信息，仅有手机号或随机数字。

二、有价值数据（keep=true）
只要手机号附近出现以下任意信息，即视为真实个人数据：

1. 姓名（如：张三、李四）
2. 身份证号
3. 地址
4. 学校 / 班级 / 公司 / 单位
5. 年龄 / 年级
6. 家庭关系（父亲、母亲等）
7. 密码 / 账号
8. 其他可以构成人物身份的信息

示例：
手机号 + 姓名 → true
手机号 + 姓名 + 班级 → true
手机号 + 学校 → true

三、重点说明
【手机号 + 姓名】本身就已经是有价值的个人信息，必须保留。

输出要求：
只输出 JSON：

{
"keep": true/false,
"reason": "简要说明判断原因"
}
"""

# ==========================================
# Prompt B（精简版）
# ==========================================

PROMPT_B = """
你是一个数据过滤器。

任务：
判断文本中的【目标手机号】是否属于真实个人数据。

判断规则：

keep=false：
1. 手机号与字母连在一起（如 a1881234b）
2. 内容是代码、日志、URL、系统输出
3. 只有手机号，没有任何人物信息

keep=true：
手机号附近出现以下任意信息：
- 姓名
- 身份证号
- 地址
- 学校 / 班级 / 公司 / 单位
- 年龄 / 年级
- 家庭关系
- 密码 / 账号

规则：
手机号 + 姓名 = keep=true

输出：
只返回 JSON：

{"keep": true}
或
{"keep": false}
"""
```
负向数据：
```
def fake_data():
    phone = 15883380267
    return phone, "4776310;DDC15883380267C2C23021F2040B5106;..."
```
测试结果明显是不要理由的效果好
```

```

正向数据：
```
def fake_data():
    phone = 15883380267
```
